# Contributing to FilmMatch AI

Thanks for your interest in contributing!

## Development Setup

1. Fork the repo and clone your fork.
2. Set up the backend and frontend per the README.
3. Create a feature branch: `git checkout -b feature/your-feature`

## Pull Request Process

1. Keep PRs focused â€” one feature or fix per PR.
2. Write tests for new functionality.
3. Update documentation if behavior changes.
4. Ensure CI passes before requesting review.

## Code Style

- **Python**: Follow PEP 8. Use type hints.
- **TypeScript**: Use strict mode. Prefer named exports.

## Commit Messages

Use conventional commits:
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `refactor:` code refactoring
- `test:` adding or updating tests
- `chore:` maintenance

## Reporting Issues

Use the GitHub issue templates. Include steps to reproduce for bugs.
